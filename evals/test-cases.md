# Test Cases

Use these cases to check whether the skill behaves as intended.

## Case 1: Theme Prompt Builder

User:

```text
Build me a Serenity-style research prompt for the AI infrastructure supply chain.
```

Expected behavior:

- Produce a reusable prompt, not a completed report.
- Focus on demand shocks, value-chain mapping, scarce layers, second- and third-order beneficiaries, evidence, scoring, and falsification.
- Stay general to the user-provided market theme and avoid importing unrelated case-study material.
- Avoid direct buy/sell advice.

## Case 2: Company Challenge

User:

```text
Challenge whether Company X is a real AI infrastructure bottleneck.
```

Expected behavior:

- Identify Company X's value-chain layer.
- Determine whether it controls a scarce layer, supplies one, benefits indirectly, or mostly borrows the narrative.
- List evidence required from filings, transcripts, customers, capacity, contracts, or technical sources.
- State what would falsify the thesis.

## Case 3: Second-Order Beneficiaries

User:

```text
Rank second-order beneficiaries of a surge in live-streaming traffic.
```

Expected behavior:

- Explain the streaming demand shock.
- Map downstream viewing to upstream infrastructure.
- Rank layers before companies.
- Include CDN, cloud, data centers, networking, optical modules, encoding, power, cooling, and security where relevant.
- Score candidates as research priorities, not expected returns.

## Case 4: Learning Mode

User:

```text
Teach me the method one question at a time.
```

Expected behavior:

- Ask one focused question.
- Move from trend to demand shock to value chain to scarce layer to evidence to falsification across turns.
- Avoid dumping a full report immediately.
